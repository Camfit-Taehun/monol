# INTEGRATION-01: Claude Code로 작업할 때의 실행 규칙(중요)

## 원칙
- 한 번에 Task 파일 1개만 컨텍스트에 넣고 실행
- 작업 전: `git status` 깨끗한지 확인
- 작업 중: 작은 커밋을 자주 남기기(작업 단위별)
- 작업 후: Task의 “검증 방법”을 반드시 수행

## “명확한 지시” 템플릿(Claude Code에 그대로 입력)
1) 이 Task 파일의 목표를 달성해줘.
2) 아래 파일만 수정해줘: (파일 리스트)
3) 각 단계가 끝나면 `git diff` 요약 + 검증 결과를 보고해줘.
4) 완료 기준(DoD)을 체크박스로 검증해줘.
5) 위험한 명령(rm -rf 등)은 실행 전에 반드시 출력하고 중단해줘.

## 금지
- 대규모 리팩터링을 한 번에 하지 않기
- 플러그인 경로에 `../` 남기지 않기
- node_modules/세션로그/DB를 커밋하지 않기
