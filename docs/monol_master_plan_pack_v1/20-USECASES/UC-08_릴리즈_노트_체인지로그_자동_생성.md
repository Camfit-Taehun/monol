# UC-08: 릴리즈 노트/체인지로그 자동 생성

## 가치(Why)
릴리즈마다 변경사항을 자동 분류/요약해 배포 커뮤니케이션 비용을 줄입니다.

## 범위(What)
- 커밋/세션 요약을 기반으로 CHANGELOG/Release Notes 초안을 생성

## 선행 조건/의존(Task)
- TASK-200 events(선택)

## 구현 설계/개발 플랜(How)
1) 입력:
- `git log <from>..<to>` 수집
- events에서 해당 기간 세션 요약 수집(있으면)
2) 분류:
- feat/fix/perf/docs/chore/breaking
3) 출력:
- `CHANGELOG.md`에 append
- `release-notes.md` 생성
4) events 기록: `release.notes.generated`

## 수정/생성 파일(초안)
- `monol/src/core/release/notes.ts`
- `monol/.claude-plugin/commands/release-notes.md`
- `monol/tests/release-notes.test.ts`

## 테스트 시나리오(재현 절차)
테스트:
- 분류기 단위 테스트(커밋 메시지 fixture)
- 통합: from/to 범위 지정 후 파일 생성 확인

## 완료 기준(DoD)
- [ ] 기능이 end-to-end로 재현된다
- [ ] 실패 케이스(권한/빈 데이터/네트워크 오류) 처리
- [ ] 자동 테스트 최소 1개 추가(단위/통합/스크립트)
