# UC-10: 시크릿/취약점 스캔 + 리메디에이션 제안

## 가치(Why)
시크릿 유출과 취약점 의존성을 조기에 탐지하고 안전한 수정 흐름을 제공합니다.

## 범위(What)
- diff 시크릿 탐지(훅) + npm audit 요약 + 대응 가이드 자동 생성

## 선행 조건/의존(Task)
- UC-02

## 구현 설계/개발 플랜(How)
1) diff 스캔: 고위험 패턴(AWS keys, private key block)
2) 발견 시:
- 작업 중단(옵션) 또는 경고
- .env 이동/키 회전 가이드 생성
3) npm audit high/critical 요약 + 업데이트 제안

## 수정/생성 파일(초안)
- `monol/src/core/security/scan-secrets.ts`
- `monol/src/core/security/audit.ts`
- `monol/tests/security-secrets.test.ts`

## 테스트 시나리오(재현 절차)
테스트:
- regex 탐지 단위 테스트
- audit 결과 파서 단위 테스트
