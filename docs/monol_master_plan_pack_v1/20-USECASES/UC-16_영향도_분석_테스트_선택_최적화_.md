# UC-16: 영향도 분석(테스트 선택 최적화)

## 가치(Why)
변경 영향 범위를 분석해 필요한 테스트만 실행합니다.

## 범위(What)
- 의존성 그래프 → 영향 패키지 → 테스트 선택

## 선행 조건/의존(Task)

## 구현 설계/개발 플랜(How)

2) 변경 파일 → 영향 패키지 계산
3) 영향 패키지에 대한 테스트만 실행
