# UC-24: 변경 코드 기반 단위 테스트 스캐폴드 자동 생성

## 가치(Why)
코드 변경이 있을 때 필요한 단위 테스트 뼈대를 자동 생성합니다.

## 범위(What)
- diff 분석 → 테스트 템플릿 생성

## 선행 조건/의존(Task)
- UC-02

## 구현 설계/개발 플랜(How)
1) git diff로 변경 파일/함수 감지
2) 스택별 템플릿(jest/vitest) 적용
3) 기존 파일 있으면 append/skip 정책

## 수정/생성 파일(초안)
- `monol/src/core/tests/generate.ts`
- `monol/tests/testgen.test.ts`

## 테스트 시나리오(재현 절차)
테스트:
- 생성 결과 스냅샷
- 기존 파일 append 정책 테스트

## 완료 기준(DoD)
- [ ] 기능이 end-to-end로 재현된다
- [ ] 실패 케이스(권한/빈 데이터/네트워크 오류) 처리
- [ ] 자동 테스트 최소 1개 추가(단위/통합/스크립트)
