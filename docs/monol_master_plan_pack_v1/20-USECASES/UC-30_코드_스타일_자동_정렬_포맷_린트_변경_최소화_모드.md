# UC-30: 코드 스타일 자동 정렬(포맷/린트) + 변경 최소화 모드

## 가치(Why)
코드 스타일을 자동 정렬하되, 변경량을 최소화해 리뷰 부담을 줄입니다.

## 범위(What)
- 포맷터 실행 + 불필요 변경 최소화 옵션

## 선행 조건/의존(Task)
- UC-02

## 구현 설계/개발 플랜(How)
1) 프로젝트 포맷터(prettier 등) 감지
2) `monol format` 명령 제공
3) 변경량이 큰 경우 경고 + 분리 커밋 제안

## 수정/생성 파일(초안)
- `monol/src/core/format/run.ts`
- `monol/tests/format.test.ts`

## 테스트 시나리오(재현 절차)
테스트:
- 포맷 명령 실행 스모크(샘플 파일)
- 변경량 계산 테스트
