# Planote — 구현 계획(WP0–WP7) (KO)

> 이 문서는 WP별로 *무엇*을 산출해야 하는지 정의한다.  
> `implementation-blueprints.md`는 *어떻게* 구현할지(파일 트리/스크립트/스켈레톤)를 정의한다.

## WP0 — 레포 부트스트랩 & 품질 게이트 (MUST)
### 목표
dev/build/test/lint가 갖춰진 프로젝트 스켈레톤과 최소 서버+UI를 만든다.

### DoD
- `npm run dev`로 서버+UI 실행
- `npm test` 통과
- 수동 QA를 `/plan/en/_qa_log.md`에 기록

### 수동 QA(WP0)
- dev 실행 후 UI에서 프로젝트 정보를 본다.
- 서버를 멈추고 재시작해도 포트 충돌이 없다.
- 서버가 기본적으로 127.0.0.1에만 바인딩되는지 확인한다.

## WP1 — 인덱서: 플랜 하이어라키 스캔 (MUST)
수동 QA:
- plan 아래 md 파일 추가/삭제 → 트리 업데이트
- 헤딩 레벨 추가 → 중첩 정확

## WP2 — UI: 트리 + 마크다운 프리뷰 (MUST)
수동 QA:
- XSS 스니펫 실행되지 않음
- 키보드 내비게이션 기본 동작

## WP3 — 주석: 파일/섹션 (MUST)
수동 QA:
- 주석 생성/필터/해결/삭제(소프트)
- 삭제된 주석이 trash에 존재

## WP4 — 선택 주석 + 앵커 재매칭 (MUST)
수동 QA:
- 편집 후에도 선택 하이라이트 유지
- 동일 quote 중복 시 ambiguous 선택 UI

## WP5 — 리뷰 사이클 + 번들 생성 (MUST)
수동 QA:
- 사이클 생성 → 번들 생성 → `.planote/bundles/` 산출물 확인

## WP6 — 리비전 감지 + diff 뷰 (v1 필수)
수동 QA:
- 플랜 파일 편집 → 리비전 감지 → diff 확인
- 리비전으로 주석 해결 처리

## WP7 — 업무 링크 + 롤업 (SHOULD)
수동 QA:
- 노드/주석에 업무 링크 추가
- 트리 배지 업데이트

리스크/대응은 이전 스펙을 따른다.
